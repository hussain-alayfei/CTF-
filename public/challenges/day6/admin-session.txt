A JSON Web Token is SIGNED, not ENCRYPTED — the middle section is just
base64url-encoded JSON that anyone can read with no key at all. A careless
developer stashed a private note inside one of the claims. Decode the
token (jwt.io, CyberChef, or `base64 -d`), read the "note" claim, decode
THAT too, and submit what it says. No cracking, no special tools.
Answer format: lowercase, words joined by underscores.
